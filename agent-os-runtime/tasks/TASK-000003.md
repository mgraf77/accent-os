# Build Dependency Engine

Implement the dependency resolution and cycle detection subsystem.

## Objective

Create `runtime/dependency_engine.py` which evaluates whether all prerequisite
tasks for a given task have reached `complete` status before it can proceed.

## Deliverables

- `check_dependencies(task, all_tasks)` — returns (satisfied, blocking_ids)
- `get_dependency_graph(tasks)` — returns adjacency dict
- `detect_cycles(tasks)` — DFS-based cycle detection

## Dependencies

Requires TASK-000001 (timestamp engine) and TASK-000002 (event logger) to both
be complete before this task can begin.

## Trigger

This task will not start before 2026-05-08T09:00:00Z even if dependencies are met.

## Notes

Cycle detection uses a recursive DFS with a path stack. A missing dependency
(referenced ID not in the task list) is treated as blocking (not satisfied).
