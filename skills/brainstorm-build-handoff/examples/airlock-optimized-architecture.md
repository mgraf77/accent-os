# PHASE 4 — Ralph Loop Optimization

## Pass 1 — Workflow Correctness

**Critique:**
- The HandoffValidation workflow has no timeout — if a rule hangs, session start hangs
- AuditLog write is synchronous inside the validation workflow — should be fire-and-forget to keep gate fast
- GateDecision has no expiry — a cached PASS could be replayed after state changes

**Simplify:**
- Remove "source_metadata" from HandoffPayload for MVP — not needed for rule-based checks
- Collapse FLAG and PASS into one status; add `warnings[]` array — simpler decision tree

**Improve:**
- Add 5-second timeout to total validation run; auto-FLAG (not BLOCK) on timeout
- Move AuditLog write to post-gate, non-blocking

**Delta:** Validation is now bounded in time; audit is decoupled from gate speed.

## Pass 2 — Entropy Reduction

**Critique:**
- "Claimed state vs actual state" check is underspecified — what counts as a mismatch?
- HandoffPayload inputs from different sources (WORK_IN_PROGRESS.md, session-handoff.md, git log) have no normalized format — rules will need source-specific parsers

**Simplify:**
- Normalize HandoffPayload to 4 required fields: `source`, `claimed_task`, `claimed_branch`, `claimed_wip_status` — everything else is optional metadata
- Use git and filesystem reads for "actual state" — no new state store needed

**Improve:**
- Define "state mismatch" as: claimed_branch ≠ `git branch --show-current` OR claimed_wip_status contradicts WORK_IN_PROGRESS.md
- Rules operate on normalized payload only — parsers are input adapters, not part of the rule engine

**Delta:** Payload is normalized; "state mismatch" is now precisely defined and testable.
