# mode: visual-thinking

> Slide deck + infographic + mind map. Use when sharing with a non-reader audience or building a teach-out asset.

## Audience default
intelligent non-expert OR named audience (specify in Step 0)

## Files generated
- `executive-summary.md` (the speaker-notes anchor)
- `slide-deck.md` (slide-by-slide architecture)
- `infographic.md` (visual layout spec)
- `mind-map.md` (Markmap-compatible)
- `relationships.mmd` (Mermaid graph from Step 4)
- `analogies.md` (each analogy doubles as a slide visual metaphor)

## Slide deck architecture

Default structure (15–25 slides):

```
1.  Title + 1-line provocation
2–3. Why this matters now (anchor + stakes)
4.  Core concept map (Mermaid render or hand-drawn equivalent)
5–7. Each Core concept gets one slide: definition + analogy + visual
8–10. Mechanics — the workflow / sequence / loop diagram
11–13. Edge cases and failure modes
14–16. Strategic implications (one per slide, decisions-grade)
17.  Common misconceptions corrected
18.  What you should now be able to do
19.  Q&A prompt
20.  Sources / references
```

Per-slide format in `slide-deck.md`:

```markdown
## Slide N: [title]
**Visual:** [description of what should appear visually]
**Bullets:** (3 max, ≤8 words each)
- bullet 1
- bullet 2
- bullet 3
**Speaker notes:** [2–4 sentences expanding the bullets — what the speaker actually says]
```

## Infographic layout

Single-page layout. `infographic.md` specifies:

```
HEADER BAND
  - Title
  - One-line provocation

LEFT COLUMN (40%): The mechanism
  - 4–6 stacked panels, each = one Core concept
  - Each panel: icon + name + 1-line definition + visual metaphor

CENTER COLUMN (30%): The relationship graph
  - Mermaid graph from Step 4, rendered to SVG
  - Feedback loops in bold/red

RIGHT COLUMN (30%): The strategic payoff
  - 3 stacked callouts: "If you understand this, you can ___"
  - Bottom: one risk callout

FOOTER BAND
  - Source(s)
  - "Generated by AccentOS educational-synthesis YYYY-MM-DD"
```

## When to combine with another mode
`visual-thinking` produces sharing assets. If Michael also needs to learn the topic himself first, run `deep-dive` first, then `visual-thinking` derived from the deep-dive output.

## Anti-patterns specific to visual-thinking
- Never put more than 3 bullets on a slide.
- Never use stock illustration callouts. Visual metaphors must come from the analogy library — they teach, decoration doesn't.
- Never put text on the relationship graph beyond node labels and edge type. Crowded graphs read as walls.
- Never produce a slide deck longer than 25 slides. Past that, split into a 2-part series.
- Never use color for decoration. Color = semantic meaning (red = risk/feedback loop, blue = mechanism, green = strategic payoff).
