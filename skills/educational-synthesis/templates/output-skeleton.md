# template: output-skeleton

> The folder layout for every educational-synthesis run. Used by Step 11.

## Target folder

```
/home/user/accent-os/knowledge/[topic-slug]/
```

`[topic-slug]` = kebab-case, ≤4 words, descriptive-not-generic.

Good slugs:
- `vendor-probability-model`
- `module-modes-architecture`
- `rfm-segmentation`
- `bigcommerce-vs-shopify-economics`
- `claude-tool-use-protocol`
- `hick-hyman-law`

Bad slugs (too generic):
- `vendor-stuff`
- `architecture`
- `module-thing`

## Mandatory files (every run)

| file | purpose |
|------|---------|
| `INDEX.md` | What's in this folder, last-updated, mode used, audience |
| `executive-summary.md` | 1-page TL;DR — every mode generates this |

## Mode-determined files

See each mode's `modes/*.md` for which files it produces. The complete possible set:

```
deep-dive.md

slide-deck.md
infographic.md

concept-glossary.md
analogies.md
relationships.mmd
faq.md

risks-limitations.md
```

## INDEX.md schema (per topic)

Every `[topic-slug]/INDEX.md` follows this structure:

```markdown
# [Topic name] — knowledge artifact

**Created:** YYYY-MM-DD
**Mode:** [active mode]
**Audience:** [calibration]
**Source(s):** [file paths or URLs]
**Originating skill:** [if invoked from skill-forge / analysis-snapshot, name it]

## Files

- `executive-summary.md` — [1-line description]
- `deep-dive.md` — [1-line description]
- `notebooklm-prompt.md` — [1-line description]
- ...

## Re-synthesize

```
"Re-do educational synthesis on [topic] in [mode] mode"
```

## Cross-references

- Companion AccentOS docs: [paths if any]
- Related knowledge artifacts: [/knowledge/other-slug/ if any]
- Related skills: [skills/X/ if any]
```
