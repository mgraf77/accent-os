# prompt: infographic

> Single-page visual layout spec for tools like Canva, Figma, or Adobe Illustrator. The output is structural — sections, hierarchy, data callouts — not a styled file.

## How to use

1. Run `educational-synthesis` in `visual-thinking` mode.
2. Open `/home/user/accent-os/knowledge/[topic-slug]/infographic.md`.
3. Hand to a designer OR import structure into Canva/Figma and add visual styling.

## Template for `infographic.md`

```markdown
# Infographic — [topic name]

**Format:** single-page (8.5×11 portrait OR 1080×1920 social vertical OR custom)
**Audience:** [calibration]
**Print/digital:** [either]
**Designer notes:** [color palette, font directives if any — else "default AccentOS palette"]

---

## Layout grid

```
┌─────────────────────────────────────────────────┐
│              HEADER BAND (10%)                  │
│  Title + 1-line provocation                     │
├──────────────┬──────────────┬───────────────────┤
│   LEFT COL   │  CENTER COL  │    RIGHT COL      │
│    (40%)     │    (30%)     │      (30%)        │
│              │              │                   │
│  THE         │  THE         │   THE             │
│  MECHANISM   │  RELATIONSHIP│   STRATEGIC       │
│              │  GRAPH       │   PAYOFF          │
│              │              │                   │
│              │              │                   │
├──────────────┴──────────────┴───────────────────┤
│              FOOTER BAND (5%)                   │
│  Source(s) + generated-by tag                   │
└─────────────────────────────────────────────────┘
```

## Section specifications

### Header band

**Content:**
- Title (≤6 words, sans-serif, bold)
- Provocation line (≤14 words, italic, gray)

**Example:**
```
RFM SEGMENTATION
Why your VIP list is lying to you about half your customers
```

### Left column — The mechanism (40% of body)

4–6 stacked panels, each = one Core concept.

**Per panel:**
```
[icon: 32px, semantic — gear / scale / clock / etc.]
[concept name: 16pt bold]
[1-line definition: 11pt regular]
[mini visual metaphor — small diagram or analogy illustration]
```

Stack vertically. Numbered 1 to N at top-left of each panel.

### Center column — The relationship graph (30% of body)

**Content:**
- Mermaid graph from `relationships.mmd`, rendered to SVG
- Core concepts as larger nodes
- Supporting concepts as smaller nodes
- Edge labels visible (verbs)
- Feedback loops in red/dotted

**Caption (below graph):** "How the pieces interact" + 1-line interpretation key (e.g., "Solid = causation, dotted = feedback")

### Right column — The strategic payoff (30% of body)

3 stacked callouts:

**Callout format:**
```
[banner-style header: "If you understand this, you can ___"]
[3-line strategic implication]
[icon at bottom]
```

Below the 3 callouts, one risk callout in red:
```
[banner: "WHERE IT BREAKS"]
[2-line failure mode + mitigation]
```

### Footer band

**Content:**
- "Source: [primary source]"
- "Companion artifact: /knowledge/[topic-slug]/"
- "Generated by AccentOS educational-synthesis · YYYY-MM-DD"

Small (8pt), gray.

## Color semantics

- Header band: dark navy or AccentOS primary
- Left col panels: white background, navy text
- Center col graph: navy nodes, gray supporting nodes, red feedback edges
- Right col callouts: green (positive payoff) or red (risk)
- Footer band: light gray

## Typography hierarchy

| element | size | weight |
|---------|------|--------|
| Title | 36pt | Bold |
| Section header | 18pt | Bold |
| Concept name | 16pt | Bold |
| Body text | 11pt | Regular |
| Caption | 9pt | Italic |
| Footer | 8pt | Regular |

## Calibration to topic shape

- **Mechanism-heavy topic** (e.g., probability model): expand left column to 50%, shrink right column to 20%
- **System-heavy topic** (e.g., feedback loops): expand center column to 50%
- **Strategy-heavy topic** (e.g., vendor decision framework): expand right column to 40%

Default 40/30/30 split is calibrated for typical mechanism + system + strategy balance.

## Anti-patterns specific to infographic

- Never use stock photo decoration. Every visual must teach.
- Never use icons that don't have semantic meaning to the concept.
- Never overload the relationship graph. >15 nodes on a single-page infographic = unreadable. Split or simplify.
- Never use more than 3 colors beyond palette neutrals. Color = semantic; too many colors = decoration.
- Never produce an infographic that requires the reader to follow specific reading order. Single-page infographic = scannable from any entry point. Reading order belongs on slide decks.
- Never duplicate the deep-dive content in infographic prose. Infographic = sparse + visual. Prose density of `deep-dive.md` is the wrong shape here.
```
