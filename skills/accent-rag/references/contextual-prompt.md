# Contextual prompt · ingest-time

Generated by Claude Haiku 4.5 (`claude-haiku-4-5-20251001`). Cost: ~0.0002 per chunk.

## System prompt
```
You write a single-sentence neutral context prefix so a chunk can be retrieved standalone.
Format: "From <doc title>, in <section>: <topic>."
Be specific. No quotes, no preamble.
```

## User prompt template
```
<document title="{TITLE}">
{FULL DOC BODY, truncated to 12K tokens if longer}
</document>

<chunk>
{CHUNK BODY}
</chunk>

Write the context prefix:
```

## Expected output
A single sentence, ≤ 240 chars, of the form:
> From Visual Comfort vendor playbook, in Terms snapshot: discusses Premier dealer tier 50% discount and FFA $1500 freight threshold.

## Why this works
The first 60 tokens of any chunk's `searchable` text overwhelmingly drive both BM25 (because dominant tokens) and embedding similarity (because embedding pooling weights early tokens slightly higher). A focused context prefix reliably surfaces the right chunk for a query about "Visual Comfort discount terms" even when the chunk body never says the word "discount" explicitly.

## Cache strategy

For multi-chunk docs (>5 chunks), wrap the document in an Anthropic prompt cache block. The whole-doc portion is cacheable for 5 minutes; each chunk-specific user message reuses the cached prefix. Saves ~80% on contextualization cost for any doc with ≥5 chunks.

## Fallback
When `aos-api` (Anthropic key) isn't set, BUILD-RAG and OS-RAG both fall back to a deterministic prefix:
> From {doc title}: {first non-empty line, 160 chars max}

This is enough signal for BM25 retrieval; vector recall degrades by ~10% but the system stays functional.
