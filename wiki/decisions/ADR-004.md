# ADR-004: File split strategy (index.html + js/*.js)

**Status**: Locked  
**Date**: 2026-05 (v6.10.12 file split)

## Decision

Keep index.html as the shell (auth, layout, shared utilities, inline JS for early-loaded modules). Extract larger/later-loaded modules to `js/<module>.js` files loaded via `<script src>` after the inline block.

## Context

AccentOS grew to 800KB+ as a single index.html. Claude Code edit operations on large files are slow and error-prone. The 900KB trigger for file splits was established.

## Rationale

- Each `<script src>` file is loaded globally — no import/export needed (ADR-002)
- Module functions (e.g., `marketing(el)`) are globally accessible, so the goTo() dispatcher works transparently
- Extraction done via awk/sed (for large blocks), Edit tool (for small changes)
- BUILD_INTELLIGENCE lesson: `Edit ≤30 lines, awk/sed ≥100 lines`
- True lazy-load deferred: adds 50+ LOC infrastructure + race condition risk with cross-module globals

## Split pattern

```
index.html          ← shell: auth, CSS, layout, goTo, PAGE_META, inline JS for early modules
js/module.js        ← extracted at >900KB trigger; loaded via <script src>
```

New modules: write to `js/<name>.js`, add `<script src>` at bottom of index.html, add 4 shell touchpoints (sidebar entry, PAGE_META, pages dispatcher, hydrate call).

## Consequences

- `let CUSTOMERS = []` in a module file still attaches to window — cross-module reads work transparently
- File size cap: ~900KB per file before triggering next split
- Ordering matters: modules that depend on shared utils must load after those utils

## Alternatives considered

- ES modules with import/export: rejected (would require rewriting all cross-module references)
- Lazy-load on tab activation: deferred (race condition risk, 50+ LOC overhead)
