# ADR-002: Vanilla JS, no framework

**Status**: Locked  
**Date**: 2026-04 (early AccentOS build)

## Decision

AccentOS is built in vanilla JavaScript with no build step, no bundler, no framework (no React, Vue, Angular, Svelte).

## Context

AccentOS is a single-developer (Claude Code + Michael) project targeting a small internal team. Must deploy to Cloudflare Pages as static files without a build pipeline.

## Rationale

- Zero build step: `git push` deploys immediately via Cloudflare Pages
- No dependency management: no npm audit, no version conflicts, no node_modules
- All module files are plain `<script src="js/module.js">` — trivially auditable
- Vanilla DOM manipulation is sufficient for AccentOS's UI complexity
- Claude Code can write, edit, and reason about vanilla JS without framework-specific knowledge
- BUILD_INTELLIGENCE lesson: compact-CRUD modules ship in 5–8 min per module with vanilla JS pattern

## Consequences

- No component reuse abstractions (each module is self-contained ~200–400 LOC)
- Cross-module communication via global variables (CUSTOMERS, DEALS, etc.)
- No TypeScript — type safety via code conventions and review
- File size managed by extraction to js/*.js (ADR-004)
- MODULE_REGISTRY refactor is a future polish item (reduce 4 shell touchpoints to 1)

## Alternatives considered

- React: rejected (build step, overkill for CRUD modules)
- Svelte: rejected (build step, less Claude Code familiarity)
- Web Components: considered; deferred (adds complexity without clear payoff)
