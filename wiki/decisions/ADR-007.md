# ADR-007: Wiki-first RAG (Karpathy pattern)

**Status**: Locked  
**Date**: 2026-05-06 (v6.11.1)

## Decision

Implement knowledge retrieval using Andrej Karpathy's "LLM Wiki" pattern: curated markdown pages in `wiki/` as the primary grounding layer. pgvector/BM25 live-RAG is an optional secondary path. BUILD-RAG scripts are dev tools only.

## Context

AccentOS's Ask the Engine had no grounding — it was a blank Claude API call. Adding domain knowledge grounding would dramatically improve answer quality for vendor scoring, lighting spec, and SOP queries. Two options: (1) maintain a curated wiki, or (2) build a live RAG pipeline with pgvector.

## Rationale

- **Zero infra cost**: wiki/ = static files on Cloudflare Pages; no Supabase schema changes needed
- **Zero latency overhead**: JS `fetch(wiki/concepts/slug.md)` is ~50ms vs ~200ms for embed + pgvector query
- **Human-readable**: Michael can read, edit, and audit every knowledge page directly
- **Explainable**: "Grounded on [rubric-rebates]" citation links to a readable page, not an opaque chunk
- **Offline-safe**: wiki pages are cached by browser; pgvector needs live Supabase
- **Confidence tracking**: frontmatter `confidence: low` flags pages needing review; no equivalent in pgvector chunks
- pgvector path (M42/M43) remains available as secondary path when wiki has no hit

## Three-layer architecture

```
wiki/ primary → pgvector secondary (optional) → pure Claude fallback
```

## Consequences

- All knowledge must be manually curated into wiki pages (no auto-ingest)
- wiki_seed.py auto-generates module + vendor pages from source files (reduces manual burden)
- wiki/log.md + wiki/hot.md provide session continuity without reading all source files
- sendChat() modified: term overlap → fetch wiki → inject → fallback to pgvector
- BUILD-RAG scripts: rag_build_index.py gives wiki chunks 1.3× BM25 boost

## Alternatives considered

- Pure pgvector RAG: rejected (infra cost, M42/M43 required, opaque chunks)
- No grounding: rejected (hallucination risk for vendor-specific questions)
- External knowledge base (Notion, Confluence): rejected (another tool, another cost, no code integration)
