# ADR-003: Cloudflare Pages for hosting

**Status**: Locked  
**Date**: 2026-04 (early AccentOS build)

## Decision

Deploy AccentOS frontend to Cloudflare Pages. Deploy any server-side functions as Cloudflare Workers.

## Context

AccentOS is a static SPA (single-page app). Auth is handled by Supabase. No server-side rendering needed. Must be accessible from mobile (showroom floor) and desktop.

## Rationale

- Free tier: unlimited bandwidth, 500 builds/month, global CDN
- Zero-config deployment: connect GitHub repo → push = deploy
- Cloudflare Workers available for edge functions (e.g., Windward proxy, wiki search shim)
- Global CDN: < 50ms TTFB from Wichita, KS (target user base)
- Custom domain support on free tier

## Consequences

- All sensitive keys (Supabase URL, Supabase API key, Anthropic API key) entered by user at runtime via Settings — never committed to repo
- No server-side rendering, no SSR-dependent features
- Workers needed for: API key proxying (if needed), Windward S5WebAPI integration (Track 6.11)
- Build pipeline is: git push → Cloudflare Pages → deploy (< 60 seconds total)

## Alternatives considered

- Vercel: considered; Cloudflare has better edge network and free Workers tier
- GitHub Pages: rejected (no edge functions, no Workers)
- AWS S3 + CloudFront: rejected (operational overhead)
